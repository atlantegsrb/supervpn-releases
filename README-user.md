# superVPN — руководство пользователя

## Что скачать

Перейдите на страницу релизов и скачайте файл под вашу систему:

| Система | Файл |
|---|---|
| Windows (любая, в т.ч. RDP / виртуальная машина) | `supervpn-client-gui-windows-amd64.exe` |
| Windows (с видеокартой, не RDP) | `supervpn-client-gui-windows-fyne-amd64.exe` |
| macOS Apple Silicon (M1/M2/M3) | `supervpn-client-gui-darwin-arm64` |
| macOS Intel | `supervpn-client-gui-darwin-amd64` |
| macOS (оба в одном, .app) | `superVPN-macos.zip` |

> **Если не знаете какой выбрать на Windows** — берите `windows-amd64.exe` (первый вариант). Он работает везде.

---

## Windows

1. Скачайте `.exe` и запустите **от имени администратора** (правая кнопка → «Запуск от имени администратора»).  
   Права администратора нужны один раз при первом запуске, чтобы автоматически установить сетевой драйвер.

2. Заполните поля:
   - **Server** — адрес сервера, который выдал администратор (например `vpn.example.com:5555`)
   - **Login** / **Password** — ваши учётные данные
   - **Hub ID** — номер хаба (обычно `1`, уточните у администратора)

3. Нажмите **Connect**.

4. Индикатор в верхней части окна станет **зелёным** — соединение установлено.

При следующем запуске настройки восстановятся автоматически — просто нажмите Connect.  
Или поставьте галочку **Auto-connect on startup** (вкладка Advanced) — и приложение подключится само.

---

## macOS

1. Распакуйте `superVPN-macos.zip` и перенесите `superVPN.app` в папку «Программы».

2. При первом запуске macOS может сказать «невозможно проверить разработчика» — откройте  
   **Системные настройки → Конфиденциальность и безопасность** и нажмите «Открыть».

3. Приложение попросит пароль (или sudo) при подключении — это нормально, нужно для создания сетевого интерфейса.

4. Заполните те же поля (Server, Login, Password, Hub ID) и нажмите **Connect**.

---

## Что означают поля

| Поле | Что вводить |
|---|---|
| Server | `хост:порт` — выдаёт администратор |
| Login | ваш логин |
| Password | ваш пароль |
| Hub ID | номер сети (обычно `1`) |
| Mode | оставьте `auto` |
| Transport | оставьте `auto` |

---

## Индикатор статуса

В верхней части окна есть цветной кружок:

| Цвет | Значение |
|---|---|
| ⚫ Серый | Не подключено |
| 🟡 Жёлтый | Подключение... |
| 🟢 Зелёный | Подключено |
| 🔴 Красный | Ошибка / переподключение |

---

## Сохранение настроек

Нажмите **Save…** и сохраните файл `.toml` в удобное место.  
При следующем запуске приложение само найдёт и загрузит последний использованный файл.

---

## Вкладка Advanced — полезные настройки

| Настройка | Описание |
|---|---|
| **Minimize to tray on close / minimize** | При закрытии или сворачивании окна приложение уходит в трей (значок в области уведомлений). Нажмите на него левой кнопкой, чтобы вернуть окно. |
| **Auto-connect on startup** | Автоматически подключается при запуске приложения, не требуя нажатия Connect. |

Остальные параметры (FEC, UDP, TLS) менять не нужно — они настроены оптимально.

---

## Мост (Bridge mode) — только Windows

Если вы используете режим моста (сетевой мост для L2-трафика), приложению нужна библиотека **Npcap**.  
Кнопка **Npcap** на вкладке Connection показывает статус:

- **Npcap ✓** (серая) — установлена, всё готово  
- **Install Npcap** (активная) — не установлена; нажмите кнопку, появится мастер установки

## Частые вопросы

**Кнопка Connect не работает / пишет ошибку**  
→ Проверьте адрес сервера, логин и пароль. На Windows убедитесь, что запустили от администратора.

**После рестарта сервера пишет Connected, но интернет не работает**  
→ Нажмите Disconnect, потом Connect. Или подождите 5 секунд — клиент переподключится сам.

**На Windows пишет «TAP driver not found»**  
→ Запустите от администратора — драйвер установится автоматически.

**На macOS пишет «operation not permitted»**  
→ Введите пароль в диалоге, который появляется при подключении.

**Значок приложения виден в трее, но окно не открывается**  
→ Нажмите на значок левой кнопкой мыши.

**Клиент завис или не отвечает**  
→ Закройте и откройте заново. Настройки не потеряются.
